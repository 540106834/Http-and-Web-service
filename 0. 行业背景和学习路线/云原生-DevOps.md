## 1️⃣ 云原生概念

- **云原生（Cloud Native）**：利用云平台特性（弹性、可扩展、容器化）来开发、部署和运行应用
- 核心理念：**容器化 + 动态调度 + 微服务 + 自动化运维**

关键技术：

| 技术/概念                     | 作用                                         |
| ----------------------------- | -------------------------------------------- |
| **容器化（Docker）**          | 将应用及依赖打包，轻量、可移植               |
| **容器编排（Kubernetes）**    | 管理成百上千的容器，自动部署、伸缩、健康检查 |
| **微服务架构**                | 拆分单体应用，每个服务独立部署和升级         |
| **服务网格（Istio/Linkerd）** | 管理服务间通信、负载均衡、安全策略           |
| **CI/CD**                     | 自动化构建、测试、部署                       |
| **监控/日志/告警**            | Prometheus/Grafana/ELK，保障系统可观测性     |

------

## 2️⃣ 核心技能栈

### 基础技能

- **Linux**：文件、权限、网络、系统管理
- **网络**：TCP/IP、HTTP、DNS、负载均衡
- **Git**：版本控制、分支管理

### 容器化与微服务

- **Docker**：镜像构建、容器运行、数据卷、网络
- **Kubernetes**：Pod、Deployment、Service、ConfigMap、Secret、Ingress
- **Helm**：K8s 包管理工具，快速部署应用
- **微服务设计**：Spring Boot/Spring Cloud、服务注册与发现、熔断、网关

### 自动化运维/DevOps

- **CI/CD 工具**：Jenkins、GitLab CI、GitHub Actions
- **基础自动化**：Ansible、Terraform
- **监控与日志**：Prometheus + Grafana、ELK/EFK、Alertmanager
- **容器监控**：cAdvisor、Kube-state-metrics

### 云平台（选学）

- AWS、GCP、Azure 基础资源管理
- 云原生服务：EKS/GKE/AKS、RDS、S3/OSS

------

## 3️⃣ 学习路线推荐

### 阶段 1：基础（1–2 个月）

- Linux + 网络基础 + Git
- Docker 入门，学会构建和运行容器

### 阶段 2：容器编排与微服务（2–3 个月）

- Kubernetes 核心概念（Pod、Deployment、Service、Ingress）
- Helm 部署应用
- Spring Boot + Spring Cloud 微服务实验

### 阶段 3：DevOps 与自动化（2–3 个月）

- CI/CD 实战（Jenkins/GitLab CI）
- Ansible/Terraform 自动化部署
- 监控与日志（Prometheus + Grafana + ELK）
- 了解服务网格和安全策略

### 阶段 4：进阶与项目实践（持续）

- 云原生完整项目（微服务 + CI/CD + K8s + 监控）
- 多环境部署、弹性伸缩、故障恢复演练
- 性能优化与安全加固

------

💡 **核心思路**：

1. **先容器化** → 学会 Docker 打包运行应用
2. **再编排** → Kubernetes 管理容器、微服务部署
3. **再自动化** → CI/CD + 配置管理 + 监控告警
4. **最后云原生** → 整合云服务，保障可扩展、高可用
