# **运维/DevOps 代码学习清单**

## **1️⃣ 自动化脚本与基础编程**

**目标**：快速处理重复任务、批量管理服务器、日志分析。

| 技能       | 推荐语言    | 学习内容                           | 使用场景                                 |
| ---------- | ----------- | ---------------------------------- | ---------------------------------------- |
| Shell/Bash | Linux shell | 变量、循环、条件、函数、文件处理   | 启动/停止服务、备份、日志清理、批量操作  |
| Python     | Python3     | requests、os、subprocess、JSON处理 | API 调试、日志分析、自动化任务、运维工具 |
| PowerShell | PowerShell  | 变量、流程控制、文件操作、远程命令 | Windows 服务器管理、自动化部署           |

> **建议**：优先学 Bash + Python，覆盖 Linux 和跨平台任务。

------

## **2️⃣ 配置管理与基础即代码**

**目标**：用代码管理服务器、网络、云资源，实现基础设施自动化。

| 技能           | 推荐工具/语言   | 学习内容                                | 使用场景                           |
| -------------- | --------------- | --------------------------------------- | ---------------------------------- |
| 配置管理       | Ansible/YAML    | Playbook、Inventory、模块、变量         | 批量配置服务器、部署软件、执行任务 |
| 基础设施即代码 | Terraform/HCL   | 资源定义、模块、变量、provider          | 云资源自动化管理（AWS/Azure/GCP）  |
| 容器编排       | Kubernetes YAML | Deployment、Service、Ingress、ConfigMap | 部署微服务、自动伸缩、服务发现     |

> **重点**：掌握 YAML 语法、模板变量和模块化思维即可。

------

## **3️⃣ Web/API 调试与自动化**

**目标**：理解和调试现代 Web 服务与接口。

| 技能           | 工具/语言                       | 学习内容                                       | 使用场景                           |
| -------------- | ------------------------------- | ---------------------------------------------- | ---------------------------------- |
| HTTP 请求      | curl / httpie / Python requests | GET/POST/PUT/DELETE、Header、Body、Status Code | 测试 REST API、GraphQL、微服务接口 |
| JSON 处理      | jq / Python json                | 解析、过滤、提取字段                           | 分析 API 返回、日志数据处理        |
| 自动化接口测试 | Python / Postman scripting      | 请求链、定时任务、脚本化测试                   | 定期健康检查、接口监控             |

------

## **4️⃣ CI/CD 与自动化部署**

**目标**：用代码和配置实现自动构建、测试和发布。

| 技能             | 工具/语言                                | 学习内容                         | 使用场景                            |
| ---------------- | ---------------------------------------- | -------------------------------- | ----------------------------------- |
| CI/CD 配置       | GitHub Actions / GitLab CI / Jenkinsfile | pipeline、job、artifact、secret  | 自动化构建、测试、部署 Web/API 服务 |
| Docker           | Dockerfile、docker-compose.yml           | 镜像构建、容器管理、Volume、网络 | Web 服务容器化部署、微服务管理      |
| 证书与安全自动化 | Bash / Python                            | Let’s Encrypt 自动更新、TLS 配置 | HTTPS 自动化管理                    |

------

## **5️⃣ 监控与排错脚本**

**目标**：快速定位问题，提高服务可用性。

| 技能     | 工具/语言                    | 学习内容                     | 使用场景                       |
| -------- | ---------------------------- | ---------------------------- | ------------------------------ |
| 日志分析 | Bash + awk/sed/jq / Python   | 日志过滤、统计、告警         | 分析 Nginx、应用日志、错误排查 |
| 网络调试 | Bash/curl/tcpdump/Wireshark  | 请求抓包、端口检查、响应分析 | 网络连通性、API 性能调试       |
| 指标收集 | Python / Prometheus Exporter | 自定义指标、报警脚本         | CPU/内存/接口响应监控          |

------

## **6️⃣ 建议学习顺序**

1. **Shell/Bash 基础** → 适应 Linux 环境，快速写自动化脚本
2. **Python 基础** → API 调试、日志分析、自动化工具
3. **Docker + CI/CD** → 自动化部署、服务管理
4. **配置管理 / IaC** → Ansible、Terraform、K8s YAML
5. **高级运维脚本** → Prometheus、Grafana、抓包、日志分析

------

### 🔹 核心原则

- **不开发业务功能**：代码主要用于**自动化、排查、部署、监控**
- **脚本为主，复杂逻辑少**：理解工具和流程比写算法更重要
- **实践为王**：每天写点脚本或配置，比单纯看文档更快
